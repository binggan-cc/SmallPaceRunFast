# Drill-Down Explainer Spec 研究报告

- **主题：** AI交互式图解解释器 / 无限深度钻取工具
- **来源：** GitHub - vthinkxie/illustrated-explainer-spec
- **链接：** https://github.com/vthinkxie/illustrated-explainer-spec
- **时间：** 2026-04-27

---

## Step 1: 项目定位

- **一句话描述：** 输入一个主题，得到一张插画解释图；点击图中任意位置，AI生成"钻入"该点的下一层画面，无限深度，画风全程一致
- **核心问题：** 如何让AI图解工具实现"无限微缩"体验，且画风不崩
- **目标用户：** 教育内容创作者 / 科普作者 / 任何需要"一层层拆解"视觉解释的人

---

## Step 2: 技术栈分析

- **架构：** 单页 Web App + 后端 API（解耦，客户端不含任何prompt或key）
- **核心机制：坐标标记替代法**
  - 用户点击 → canvas在原图上合成**红色圆圈+中心点**（坐标×图像宽高）
  - 将带标记的原图发给图像模型 + prompt："钻入红圈所在位置，不要在输出中包含红圈"
  - 图像模型天然擅长处理"看图"任务，不需要解析数字坐标
- **内容寻址缓存：**
  - 首次页面 id = `hash("first" + version + normalize(query))`
  - 子页面 id = `hash("child" + version + parentId + round(x,2) + round(y,2))`
  - 相同坐标点击 → 相同子页面（缓存命中）
  - Bump version → 全量缓存失效
- **状态模型：** 有序页面数组 + 当前索引指针；点击从中间页生成新分支时，截断该点之后所有页面
- **三个UI控件：** Back / 跳转任意历史页(缩略图条) / Reset

## Step 4: 竞品对比

| 维度 | Drill-Down Explainer | 普通AI图解 | 交互式信息图 |
|---|---|---|---|
| 交互方式 | 点击图像任意位置钻入 | 文字描述生成下一张 | 多为固定热区 |
| 深度 | 理论无限 | 受限于叙事设计 | 有限层级 |
| 画风一致性 | 全程保持 | 难保持 | N/A |
| 核心创新 | 坐标→红圈标记"翻译" | 无 | 无 |

**关键洞察：** "不要让模型解析坐标数字，而是把坐标翻译成图像语言（红圈标记）"

---

## Step 5: 价值判断

- **值得研究：** 是
- **启发点：**
  1. **"翻译层"设计** — 把抽象坐标"翻译"成图像模型天然理解的视觉语言，比直接喂数字高明得多
  2. **内容寻址缓存** — 相同query/click永远返回相同页面，Back按钮零成本
  3. **Spec作为交付物** — 这个仓库存的是"规格文档"而非实现代码，可以交给任意LLM或人类实现
