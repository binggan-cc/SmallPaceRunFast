## 分镜脚本（SHOT LIST）

### 0.0 – 3.0 秒
**Medium shot · 室内卧室**

8岁男孩坐在床上，手臂向前伸展，掌心朝向桌上的一支笔。他用尽全力施展念力。面部紧绷、眼睛眯着、身体因用力而颤抖，床有细微弹动。可听到闷哼声。镜头缓慢推进到男孩面部。

---

### 3.0 – 5.0 秒
**Wide shot · 全景房间**

大全景展示完整房间。一侧是床，书桌上有笔。墙上海报、书本、散落的玩具——一个真实的小孩卧室。男孩继续对笔施力。窗帘轻微飘动。镜头锁定，有轻微的手持晃动感。笔在前景，男孩在背景中失焦。

---

### 5.0 – 7.0 秒
**Close-up · 男孩面部特写**

极端专注的表情。牙关紧咬、脸颊鼓起、眉毛压低。他发出最后一次用力的闷哼。细微的面部颤抖，过度用力的喜感。

---

### 7.0 – 8.5 秒
**Macro close-up · 桌上笔的微距特写**

寂静。笔微微向前挪动了几厘米，发出细微的刮擦声。一个微小的动作。

---

### 8.5 – 11.0 秒
**Medium shot · 男孩中景**

他注意到了微小的移动，但以为是失败。表情失落。长长地叹了口气。手臂无力垂落。他夸张地向后倒在床上，床垫压缩又弹回。他盯着天花板。

---

### 11.0 – 15.0 秒
**Crash zoom · 房子飞起的大全景**

一个静止的瞬间。然后突然快速Crash zoom向后，从男孩身边穿过卧室，撞出窗户，一个连贯的镜头运动。极速拉出到宏大外景揭示：整个房子漂浮在地面上方6米处，在街道上方的空中缓缓悬停。树下和院子在下方。细微的碎屑向下飘落。房子微微倾斜。金色阳光。定格在宽广英勇喜感的揭示镜头。

## 创作方法论

### 为什么用角色设计表作为前置？
```
[GPT Image 2] → 角色设计表（正面/侧面/背面，白底）
       ↓ 参考图
[Seedance 2.0] → 注入分镜脚本
       ↓
角色一致性 + 场景多样性
```

### 分镜脚本的关键要素

| 要素 | 说明 |
|------|------|
| **时间码** | 精确到秒，明确每镜起止 |
| **镜头类型** | Medium / Wide / Close-up / Macro / Crash zoom |
| **动作描述** | 精确到肢体/表情的细节 |
| **音效暗示** | grunt / scrape sound / silence beat（帮助生成动态节奏） |
| **情绪弧线** | 努力→失败→失落→反转惊喜 |
| **镜头运动** | push in / locked / handheld / crash zoom / pull rapidly |
| **环境细节** | 窗帘飘动 / 床垫压缩 / 碎屑飘落（生活感细节） |

## 喜感+诗意反转结构

| 段落 | 情绪 | 喜剧元素 |
|------|------|---------|
| 0-3s | 认真努力 | 过度用力的肢体 |
| 3-7s | 持续用力 | 夸张表情 / 笔微动 |
| 8.5-11s | 失落放弃 | 戏剧性倒地 |
| 11-15s | 巨大反转 | 房子飞天！诗意+喜感 |

**核心技巧：** 前面用"失败"建立观众预期，最后用"更大的成功"打破预期
